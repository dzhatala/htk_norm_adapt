      If you put the error message in the subject of this thread but not in the message then it is easy to overlook the error message. Please don't depend on others seeing the subject. For me, I often open many threads at a time before answering any of them so by the time I read the thread I have forgotten the subject. However do try to provide a meaningful subject so people can find the questions they think they can help with.

       

      Also, try to balance the amount of code. Too much code makes it difficult to find a problem. Don't expect people trying to help to find a problem in a lot of code if your intent to post a lot of code is to avoid deciding what code is relevant. However it often happens that not enough code is shown, so I certainly understand it is often not obvious about what code to show.

       

      In this situation, you say:

      "Winmm.lib" was add on the item "project property pages-Linker-General-Additional Library Directories"

      The word "Directories" is a big clue. That option is for directories. There is another option called "additional input" or something like taht, which is for library files. Try using that for the lib file.

      I was getting the same "unresolved external symbol" error message while trying to play a simple wave file with PlaySound() in a C source code file using Visual C++ 2008 Express.  Adding "Winmm.lib" to "Project Properties > Linker > Input > Additional Dependencies" as SimpleSample points out fixes the problem.  Any C project utilizing multi-media must add this dependency as well.

       

      As an alternative, you can simply add this line of code to the top of the source file:

       

      Code Snippet
      #pragma comment( lib, "Winmm.lib" )
